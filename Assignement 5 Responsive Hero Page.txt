Theory: Responsive Hero Page
🔹 Introduction

A hero section is the main banner of a website that appears at the top. It usually contains a heading, description, and image. Making it responsive ensures it looks good on all devices like mobile, tablet, and desktop.

🔹 Objective

To design a responsive landing (hero) section using Flexbox that adjusts layout based on screen size.

🔹 Concepts Used
1. Flexbox
Flexbox is a CSS layout model used to arrange elements in rows or columns.
It makes alignment and spacing easier.

Properties used:

display: flex → activates flexbox
justify-content → horizontal alignment
align-items → vertical alignment
2. Responsive Design
Makes the webpage adaptable to different screen sizes.

Used:

@media query
Changes layout when screen width is small
3. Viewport Meta Tag
<meta name="viewport" content="width=device-width, initial-scale=1.0">
Ensures proper scaling on mobile devices

🔹 How Responsiveness Works
On desktop → text and image appear side by side
On mobile → layout changes to column (one below another)

🔹 Features of Hero Page
Attractive gradient background
Call-to-action button
Image + text alignment
Mobile-friendly layout

🔹 Advantages
Improves user experience
Works on all devices
Modern web design approach

🔹 Conclusion
Responsive design is essential in modern web development. Using Flexbox and media queries helps create flexible layouts that adapt to different screen sizes effectively.
